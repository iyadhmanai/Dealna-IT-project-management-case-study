# Communication Plan

## Purpose

This plan defines how project information should be shared during the Dealna MVP build.

## Communication Goals

- Keep stakeholders aligned on scope and priorities.
- Surface risks and blockers early.
- Track milestone progress.
- Prevent uncontrolled scope changes.
- Make the project easy to explain in job applications and interviews.

## Communication Channels

| Channel | Purpose |
| --- | --- |
| Project documentation | Source of truth for requirements, roadmap, risks, and decisions |
| Backlog board | Track tasks, status, priorities, and blockers |
| Weekly status update | Summarize progress, next steps, risks, and decisions needed |
| Milestone review | Validate completed work against acceptance criteria |
| Demo script | Present the project clearly to recruiters or hiring managers |

## Meeting Cadence

| Meeting | Frequency | Participants | Purpose |
| --- | --- | --- | --- |
| Planning review | Start of project | PM, PO, DEV | Confirm scope, roadmap, and assumptions |
| Backlog refinement | Weekly | PM, PO, DEV | Clarify upcoming work and acceptance criteria |
| Status update | Weekly | PM, PO, SP | Share progress, risks, and timeline changes |
| Milestone review | End of each phase | PM, PO, DEV, QA | Validate deliverables against acceptance criteria |
| Retrospective | Every 2-3 weeks | PM, DEV, QA | Improve delivery process |

```md
- Item 2

## Risks and Issues

| Risk or Issue | Impact | Owner | Action |
| --- | --- | --- | --- |

## Decisions Needed

- Decision 1

## Timeline or Scope Changes

- Change 1
```

## Escalation Rules

Escalate immediately when:

- A Must Have feature is at risk.
- Voucher security or duplicate redemption risk is discovered.
- Timeline impact is greater than one week.
- A new compliance-sensitive category is proposed.
- A scope change affects cost, schedule, or architecture.

## Stakeholder Reporting Style

For technical stakeholders, provide implementation detail and blockers.

For business stakeholders, focus on scope, timeline, cost, risk, and decisions needed.

For hiring managers, focus on the PM process, reasoning, trade-offs, and deliverables created.
